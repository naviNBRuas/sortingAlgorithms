Block Sort

Introduction

Block sort is a sorting algorithm that works by grouping the elements into blocks and then sorting each block. This method helps reduce the number of comparisons and improve the performance of the sorting algorithm.

Method

The algorithm works by dividing the input array into blocks of size M, where M is a constant. Each block is sorted using an internal sorting algorithm, such as insertion sort or quicksort. After the blocks are sorted, they are merged together to form the final sorted array.

The size of the block M affects the performance of the algorithm. If M is too small, the algorithm will have to make many comparisons, leading to a slower sorting time. If M is too large, the blocks may not be fully sorted, leading to a longer merging time. A common choice for M is around sqrt(n), where n is the number of elements in the array.

Time Complexity

The best-case time complexity of block sort is O(n), which occurs when the input array is already sorted. The average-case time complexity is O(n log n), which occurs when the blocks are sorted randomly. The worst-case time complexity is O(n^2), which occurs when the blocks are sorted in reverse order.

The memory complexity of block sort is O(n), as the algorithm requires additional memory to store the blocks while they are being sorted and merged.

Block sort is a stable sorting algorithm, meaning that elements with the same value will retain their relative order in the final sorted array.

Pseudocode

function blockSort(array A, int M)
int n = length of A
divide A into blocks of size M
sort each block using an internal sorting algorithm
merge the blocks together to form the final sorted array
end function

Bubble Sort

Introduction

Bubble sort is a simple sorting algorithm that repeatedly steps through the list, compares adjacent elements and swaps them if they are in the wrong order. The algorithm gets its name from the way smaller elements “bubble” to the top of the list.

Method

The algorithm works by starting at the beginning of the list and comparing the first two elements. If the first element is larger than the second element, they are swapped. The algorithm then moves to the next pair of elements and repeats the process. This process is repeated for each pair of elements until the end of the list is reached.

After the first pass, the largest element will have “bubbled” to the end of the list. The algorithm then repeats the process for the remaining n-1 elements, ignoring the last element since it is guaranteed to be sorted. This process is repeated until the list is fully sorted.

Time Complexity

The best-case time complexity of bubble sort is O(n), which occurs when the input array is already sorted. The average-case time complexity is O(n^2), which occurs when the elements are randomly ordered. The worst-case time complexity is O(n^2), which occurs when the elements are sorted in reverse order.

The memory complexity of bubble sort is O(1), as the algorithm only requires a constant amount of memory to store a few variables.

Bubble sort is a stable sorting algorithm, meaning that elements with the same value will retain their relative order in the final sorted array.

Pseudocode

function bubbleSort(array A)
int n = length of A
repeat
swapped = false
for i = 0 to n-2
if A[i] > A[i + 1]
swap A[i] and A[i + 1]
swapped = true
end for
n = n - 1
until not swapped
end function

Cocktail Shaker Sort

Introduction

Cocktail shaker sort, also known as bidirectional bubble sort, is a variation of bubble sort that sorts the elements in both directions. This means that after each pass, the algorithm sorts both the highest and lowest elements in the list, effectively reducing the number of passes required to sort the entire list.

Method

The algorithm works by first checking the largest element in the list and swapping it with the last element in the list. The algorithm then moves in the opposite direction and checks the smallest element in the list, swapping it with the first element in the list. The algorithm then repeats the process for the remaining n-2 elements, ignoring the two elements that are now guaranteed to be sorted. This process is repeated until the list is fully sorted.

Time Complexity

The best-case time complexity of cocktail shaker sort is O(n), which occurs when the input array is already sorted. The average-case time complexity is O(n^2), which occurs when the elements are randomly ordered. The worst-case time complexity is O(n^2), which occurs when the elements are sorted in reverse order.

The memory complexity of cocktail shaker sort is O(1), as the algorithm only requires a constant amount of memory to store a few variables.

Cocktail shaker sort is a stable sorting algorithm, meaning that elements with the same value will retain their relative order in the final sorted array.

Pseudocode

function cocktailShakerSort(array A)
int n = length of A
int left = 0
int right = n - 1
repeat
for i = left to right
if A[i] > A[i + 1]
swap A[i] and A[i + 1]
swapped = true
end for
right = right - 1
for i = right down to left
if A[i - 1] > A[i]
swap A[i - 1] and A[i]
swapped = true
end for
left = left + 1
until not swapped
end function

Comb Sort

Introduction

Comb sort is a comparison-based sorting algorithm that is a variation of bubble sort. The algorithm works by using a gap value to compare elements that are far apart from each other, effectively reducing the number of passes required to sort the entire list.

Method

The algorithm works by first calculating the gap value between the elements to be compared. The gap value starts at n-1 (where n is the length of the array) and is gradually decreased by a shrink factor until it reaches 1.

In each iteration of the algorithm, the elements that are gap elements apart from each other are compared and swapped if they are out of order. The algorithm continues until the gap value reaches 1 and the list is fully sorted.

Time Complexity

The best-case time complexity of comb sort is O(n), which occurs when the input array is already sorted. The average-case time complexity is O(n2/2p), where p is the number of increments used to calculate the gap value. The worst-case time complexity is O(n^2), which occurs when the elements are sorted in reverse order.

The memory complexity of comb sort is O(1), as the algorithm only requires a constant amount of memory to store a few variables.

Comb sort is a stable sorting algorithm, meaning that elements with the same value will retain their relative order in the final sorted array.

Pseudocode

function combSort(array A)
int n = length of A
int gap = n
bool swapped = true
repeat
gap = floor(gap / 1.3)
if gap = 0
gap = 1
end if
swapped = false
for i = 0 to n - gap - 1
if A[i] > A[i + gap]
swap A[i] and A[i + gap]
swapped = true
end if
end for
until gap = 1 and not swapped
end function

Cube Sort

Introduction

Cube sort is a sorting algorithm that uses the three-dimensional structure of the array to sort the elements in ascending or descending order. It is a comparison-based algorithm that performs better than other algorithms in some specific cases.

Time Complexity

The time complexity of Cube Sort is O(n^3/2), where n is the number of elements to be sorted. This makes it more efficient than other algorithms in terms of time complexity.

Method

The algorithm works by dividing the array into smaller cubes and then sorting each cube individually. After sorting each cube, the cubes are merged together to form a sorted array. The merging of the cubes is done by using a comparison-based sorting algorithm like Merge Sort or Quick Sort.

Pseudocode

function cubeSort(arr)
cube_size = ceil(cuberoot(n))
for i = 0 to cube_size
sort subarray arr[i*cube_size : (i+1)cube_size-1]
for i = 1 to cube_size
merge subarray arr[(i-1)cube_size : icube_size-1] and arr[icube_size : (i+1)*cube_size-1]
return arr

Cycle Sort

Introduction

Cycle Sort is an in-place and unstable sorting algorithm that is efficient in terms of both time and space complexity. It minimizes the number of writes to the input array, making it a useful algorithm in certain scenarios where writing to memory is a bottleneck.

Time Complexity

The time complexity of Cycle Sort is O(n^2), where n is the number of elements to be sorted.

Stability

Cycle Sort is an unstable sorting algorithm, meaning that the order of equal elements is not preserved in the sorted array.

Method

The algorithm works by iterating through the array and placing each element in its correct position, reducing the number of writes to the input array. The algorithm performs a series of cycles, where in each cycle, an element is placed in its correct position.

Pseudocode

function cycleSort(arr)
writes = 0
for cycle_start = 0 to n-2
item = arr[cycle_start]
pos = cycle_start
for i = cycle_start + 1 to n-1
if arr[i] < item
pos += 1
if pos == cycle_start
continue
while item == arr[pos]
pos += 1
item, arr[pos] = arr[pos], item
writes += 1
while pos != cycle_start
pos = cycle_start
for i = cycle_start + 1 to n-1
if arr[i] < item
pos += 1
while item == arr[pos]
pos += 1
item, arr[pos] = arr[pos], item
writes += 1
return arr

Exchange Sort

Introduction

Exchange sort, also known as swapping sort, is a simple sorting algorithm that repeatedly steps through the list to be sorted, compares each pair of adjacent elements and swaps them if they are in the wrong order.

Method

The algorithm works by iterating through the list, comparing each pair of adjacent elements and swapping them if they are in the wrong order. This process is repeated until the list is sorted.

Time Complexity

The time complexity of exchange sort is O(n^2) in the worst case, as it requires n-1 comparisons for the first pass, n-2 comparisons for the second pass, and so on, until 1 comparison in the last pass.

The memory complexity of exchange sort is O(1), as it only requires a constant amount of memory to store temporary variables for swapping elements.

Exchange sort is a stable sorting algorithm, meaning that elements with the same value will retain their relative order in the final sorted array.

Pseudocode

function exchangeSort(array A)
for i = 0 to n - 2
for j = i + 1 to n - 1
if A[i] > A[j]
swap A[i] and A[j]
end if
end for
end for
end function

Gnome Sort

Introduction

Gnome sort is a comparison-based sorting algorithm that is similar to the insertion sort algorithm. The algorithm is named after the garden gnomes, who sort objects by repeatedly comparing and swapping elements that are out of order.

Method

The algorithm works by iterating over the list of elements and comparing each element with its adjacent element. If the elements are out of order, they are swapped. The iteration continues until all elements are sorted.

Time Complexity

The best-case time complexity of gnome sort is O(n), which occurs when the input array is already sorted. The average-case time complexity is O(n^2), which occurs when the elements are randomly ordered. The worst-case time complexity is O(n^2), which occurs when the elements are sorted in reverse order.

The memory complexity of gnome sort is O(1), as the algorithm only requires a constant amount of memory to store a few variables.

Gnome sort is a stable sorting algorithm, meaning that elements with the same value will retain their relative order in the final sorted array.

Pseudocode

function gnomeSort(array A)
int i = 1
while i < length of A
if A[i-1] <= A[i]
i = i + 1
else
swap A[i] and A[i-1]
i = i - 1
end if
end while
end function

Heapsort

Introduction

Heapsort is a comparison-based sorting algorithm that can be thought of as an improved version of selection sort. The algorithm works by dividing the input into a max heap and a sorted array, and repeatedly removing the maximum element from the max heap and adding it to the sorted array.

Method

The algorithm starts by building a max heap from the input array. The max heap is then repeatedly reduced by swapping the maximum element with the last element in the heap, and then “sifting down” the new root element to restore the max heap property.

Time Complexity

The time complexity of heapsort is O(n log n), where n is the number of elements in the input array.

Pseudocode

function heapSort(array A)
int n = length of A
buildMaxHeap(A, n)
for i = n - 1 down to 1
swap A[0] and A[i]
n = n - 1
maxHeapify(A, 0, n)
end for
end function

function buildMaxHeap(array A, int n)
for i = floor(n / 2) down to 0
maxHeapify(A, i, n)
end for
end function

function maxHeapify(array A, int i, int n)
int left = 2 * i + 1
int right = 2 * i + 2
int largest = i
if left < n and A[left] > A[i]
largest = left
end if
if right < n and A[right] > A[largest]
largest = right
end if
if largest != i
swap A[i] and A[largest]
maxHeapify(A, largest, n)
end if
end function

In-Place Merge Sort

Introduction

In-Place Merge Sort is a comparison-based sorting algorithm that sorts an array by dividing it into smaller sub-arrays, sorting these sub-arrays, and then merging them back together. Unlike other sorting algorithms, In-Place Merge Sort does not require any additional memory to store the sub-arrays, as it sorts the array in place using a series of swaps.

Method

The algorithm begins by dividing the input array into sub-arrays of size 1, which are considered to be sorted. The sub-arrays are then merged back together two at a time, with the merging process taking care of sorting the elements.

Time Complexity

The best-case time complexity of In-Place Merge Sort is O(n log n), which occurs when the input array is already sorted. The average-case time complexity is O(n log n), as the merging process takes O(n) time and is repeated log n times. The worst-case time complexity is O(n^2), which occurs when the elements are sorted in reverse order.

The memory complexity of In-Place Merge Sort is O(1), as the algorithm does not require any additional memory to store the sub-arrays.

In-Place Merge Sort is a stable sorting algorithm, meaning that elements with the same value will retain their relative order in the final sorted array.

Pseudocode

function inPlaceMergeSort(array A, int l, int r)
if l < r
int m = (l + r) / 2
inPlaceMergeSort(A, l, m)
inPlaceMergeSort(A, m + 1, r)
inPlaceMerge(A, l, m, r)
end if
end function

function inPlaceMerge(array A, int l, int m, int r)
int i = l
int j = m + 1
while i <= m and j <= r
if A[i] <= A[j]
i = i + 1
else
swap A[i] with A[j]
j = j + 1
m = m + 1
end if
end while
end function

IntroSort

Introduction

IntroSort is a hybrid sorting algorithm that uses a combination of quicksort, heapsort, and insertion sort. The algorithm starts by using quicksort, but switches to heapsort if the recursion depth exceeds a certain limit. If the array is nearly sorted, the algorithm switches to insertion sort for better performance.

Method

The algorithm starts by dividing the input array into smaller sub-arrays using the quicksort method. If the recursion depth exceeds a certain limit, the algorithm switches to heapsort. If the sub-array becomes nearly sorted, the algorithm switches to insertion sort for better performance.

The algorithm continues until the entire input array is sorted.

Time Complexity

The average-case time complexity of IntroSort is O(n log n), making it a highly efficient sorting algorithm. The worst-case time complexity is O(n log n), which occurs when the input array is nearly sorted or when the recursion depth is limited.

The memory complexity of IntroSort is O(log n), as the algorithm uses a small amount of memory to store the recursion depth and other variables.

IntroSort is a stable sorting algorithm, meaning that elements with the same value will retain their relative order in the final sorted array.

Pseudocode

function IntroSort(array A, int l, int r)
int n = r - l + 1
if n <= 16
insertionSort(A, l, r)
return
end if
int maxDepth = 2 * log2(n)
IntroSortHelper(A, l, r, maxDepth)
end function

function IntroSortHelper(array A, int l, int r, int depth)
int n = r - l + 1
if n <= 16
insertionSort(A, l, r)
return
end if
if depth = 0
heapSort(A, l, r)
return
end if
int p = partition(A, l, r)
IntroSortHelper(A, l, p - 1, depth - 1)
IntroSortHelper(A, p + 1, r, depth - 1)
end function

Library Sort

Introduction

Library sort is a comparison-based sorting algorithm that is a variation of merge sort. It is used to sort data in libraries because it has a very low worst-case time complexity.

Method

The algorithm starts by dividing the input array into smaller subarrays and sorting each subarray individually. The sorted subarrays are then merged back together to form a single sorted array.

The algorithm uses a binary search to find the correct location for each element, which allows it to have a lower worst-case time complexity than other sorting algorithms such as quicksort.

Time Complexity

The best-case time complexity of library sort is O(n log n), which occurs when the input array is already sorted. The average-case time complexity is O(n log n), as the binary search used to find the correct location for each element takes O(log n) time. The worst-case time complexity is O(n log^2 n), which is significantly better than the worst-case time complexity of O(n^2) for other sorting algorithms such as bubble sort.

The memory complexity of library sort is O(n), as the algorithm requires additional memory to store the subarrays while they are being sorted and merged back together.

Library sort is a stable sorting algorithm, meaning that elements with the same value will retain their relative order in the final sorted array.

Pseudocode

function librarySort(array A)
divide A into subarrays
sort each subarray individually
merge the sorted subarrays back together
end function
